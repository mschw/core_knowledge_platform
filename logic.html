Business-Logic layer¶

This site describe the implementation of the business logic in the system.

The business logic module contain multiple sub-modules:

Access¶

This module is used to perform access verification based on user objects that need certain priviledges.

core_web_service.business_logic.access.check_priviledges_for_editor(user)¶

Validate if a user still fulfills the requirements to be an editor.

If not the user will be deleted from the editor group.

core_web_service.business_logic.access.check_priviledges_for_referee(user)¶

Validate if a user still fulfills the requirements to be a referee.

If not the user will be deleted from the referee group.

core_web_service.business_logic.access.user_in_group_for_publication(user, publication)¶

Return true if the user is in a group refereeing the publication, false else.

core_web_service.business_logic.access.validate_access_to_peer_review_for_user(peerreview, user)¶

Return true if a user may see the peer review, false otherwise.

A user can see a peerreview if the publication is publicly accessible. If it is still in review only an editor or the peerreviewer may see the peer review.

Arguments:
peerreview: a peerreview object. user: a user object.
Returns:
True if user is editor fo publication, false otherwise.
core_web_service.business_logic.access.validate_editor_for_publication(user, publication)¶

Return true if the user is an editor and the publication is being reviewed by a group the user is an editor of.

core_web_service.business_logic.access.validate_referee_for_publication(user, publication)¶

Return true if a user is a referee.

core_web_service.business_logic.access.validate_referee_or_editor(user, papergroup_id)¶

Return true if a user is an editor/referee in a special group, false otherwise.

core_web_service.business_logic.access.validate_user_is_editor(user)¶

Return true if the user is an editor.

core_web_service.business_logic.access.validate_user_is_peerreviewer(peerreview, user)¶

Return true if the user has written the peer review, false otherwise.

core_web_service.business_logic.access.validate_user_is_referee(user)¶

Return true if the user is a referee.

core_web_service.business_logic.insert.insert_bibtex_publication(bibtex, owner)¶

Will insert a publication from a bibtex string.

This function will parse the bibtex format and perform the following transformations: - Keywords will be transformed into keywords. - Each fields not suitable for the publication table will be inserted into a

key-value-pair into the furtherfields table.
Arguments:
bibtex: the bibtex for an entry. owner: the user object that will insert the publication.

Search¶

This module implements the search functionality of the service.

core_web_service.business_logic.search.build_query(search_query)¶

Build the query for a given dictionary.

Will create a Q object for each query and return a list of these queries.

Attributes:
search_query: a dictionary with search_term: search_value pairs.
Returns:
query: a list of Q-objects containing the query.
Raises:
AttributeError: when a search_value is not present for a search_term.
